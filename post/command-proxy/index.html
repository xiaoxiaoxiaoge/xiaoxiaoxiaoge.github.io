发现有些同学在浏览器使用网络代理，但对于命令行模式下的网络代理使用不是很了解，这里简单介绍下。

export http_proxy=http://proxyAddress:port

当前命令生效

把export http_proxy=http://proxyAddress:port放到profile中

4层tcp代理使用

是不是我们使用7层网络代理就能解决常见的问题了呢，遗憾的发现并不是这样，举一个例子。

你需要通过运维开通的socks代理获取现网日志时。

http_proxy=socks5://172.22.97.17:10000 wget http://172.22.97.17/logs/MicroService/ist4iflypri/see.loa
解析代理服务器 URL socks5://172.22.97.17:10000 时发生错误：不支持的协议类型 “socks5”。

每个工具对于代理的支持情况不同，会导致以上的问题，那有没有一种4层网络代理工具，可以代理一切的tcp请求呢？

这里介绍下proxychains，类似的工具很多，比如tsocks等。

安装

https://github.com/haad/proxychains#installation

配置

在配置文件/usr/local/homebrew/etc/proxychains.conf的最后一行配置以下内容

socks5  172.16.82.181 10000

使用
